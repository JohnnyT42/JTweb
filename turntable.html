Automated Model Railway Turntable

I wanted an automatic turntable and tried several systems before settling with DCC Interfaces stepper motor system. This works well, I decided that the Peco LK-55 TT needed some sound to give it weight and bulk, even if incorrect in terms of the prototype! So as I use DCC++ EX and JMRI on a Raspberry Pi4B along with Dr. Geoff Bunzaâ€™s Arduino sensor project I worked out a system using hall effect sensors. Two on each exit, one on the exit track and one unseen below the baseboard at bridge height. These trigger DCC accessory commands (turnouts in JMRI) which cause the sounds and signal to operate and enable the timing for the loco throttle control within JMRI Jython scripts.

One problem to overcome was due to my exit roads not being equally distanced so I could not use a single sound file due to timing. So I purchased a SparkFun Polymorphic Wav Trigger that allows simultaneous playing of Wav files. I have two sound files one is a start-up and continuous turning sound the second is the squeaky braking stopping sound. By using the sensors and script instructions I have the first sound start when the script starts and then the first sound stops and the second starts when the bridge reaches its desired exit. The two sounds play overlapped by about 300ms to give an impression of continuous sound. It works really well and achieves the desired effect.

The main TT sound is a recording of an actual TT, the enhanced start-up and squeaky braking sounds are made from several free sound effects and mixed using Audacity on Windows 10.

I have 14 exits (7 front and 7 back) and 14 virtual buttons on my JMRI Panel Pro Control Panel. Each one triggers a script to go to a desired exit along with the sound and led signal. Each of these smaller scripts can then be incorporated within larger jython scripts allowing overall automation of my layout. The accessory decoder I use is the Arcomora kit.
